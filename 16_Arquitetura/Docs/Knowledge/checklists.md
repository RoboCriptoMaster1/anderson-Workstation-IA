# Checklists

## Objetivo

Definir todos os checklists oficiais da Workstation IA.

Os checklists representam os critérios mínimos de qualidade utilizados pelo Cortex antes de considerar qualquer tarefa concluída.

Nenhuma entrega será considerada finalizada sem validação.

---

# Filosofia

Construir.

↓

Validar.

↓

Documentar.

↓

Entregar.

A validação é parte da implementação.

# Checklist Geral

Antes de finalizar qualquer tarefa verificar.

- Objetivo atendido.

- Arquitetura respeitada.

- Código limpo.

- Documentação atualizada.

- Testes executados.

- Segurança validada.

- Performance analisada.

- Versionamento atualizado.

# Cortex

Antes de atualizar o Cortex.

- Novo conhecimento validado.

- Memory revisada.

- RAG atualizado.

- Índices reconstruídos.

- Documentação sincronizada.

---

# Critérios de Aprovação

Uma entrega somente será considerada concluída quando.

- Todos os checklists forem aprovados.

- Nenhum erro crítico permanecer.

- Toda documentação estiver atualizada.

- O Cortex possuir conhecimento da alteração.

# Boas Práticas

- Nunca ignorar um checklist.

- Automatizar sempre que possível.

- Revisar periodicamente.

- Criar novos checklists quando necessário.

- Versionar alterações.

---

# Padrão Oficial

Todos os módulos da Workstation IA deverão utilizar os checklists definidos neste documento.

Nenhuma funcionalidade poderá ser considerada pronta sem aprovação formal dos critérios estabelecidos pelo Cortex.
